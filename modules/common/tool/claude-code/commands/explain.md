# コード説明

**対象**: $ARGUMENTS

> - **ファイルパス指定**: そのファイル全体の構造と主要な処理を説明する
> - **関数名・箇所指定**: その関数・クラスの動作を詳しく説明する
> - **処理の説明**（例: 「認証フロー」「データ変換処理」）: 該当箇所を探して説明する
> - **未指定**: 直前の会話コンテキストの対象を説明する

## 制約

- **コードの変更は行わない** — 読み取りと説明のみ
- ユーザーの知識レベルに合わせた粒度で説明する
- 推測部分（コメントがなく意図が不明な箇所）は明示的に「推測」と断る

## 調査プロセス

1. **対象の特定** — 指定された箇所を読み取る
2. **依存関係の把握** — インポート・呼び出し元・呼び出し先を追う
3. **git 調査**（必要な場合）— `git log` / `git blame` で変更経緯・意図を補完する
4. **説明の構成** — 全体像 → 詳細 → 注意点の順で組み立てる

## 説明の観点

- **何をするか** — 処理の目的・責務
- **どう動くか** — 処理フロー・アルゴリズム・データの流れ
- **なぜこうなっているか** — 設計上の意図・制約・背景（読み取れる範囲で）
- **呼び出し関係** — どこから呼ばれ、何を呼んでいるか
- **注意点** — 副作用・状態変化・エラーハンドリングの特異な箇所

## 出力形式

### 概要

[2〜3行で「何のためのコードか」を説明]

### 処理フロー

[ステップごとに何が起きるかを説明。コードの引用（ファイル名:行番号）を交えて具体的に示す]

### 依存関係

[主要な呼び出し元・呼び出し先・外部依存を示す]

### 注意点・補足

[副作用・例外ケース・わかりにくい実装上の意図など]

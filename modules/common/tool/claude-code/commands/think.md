# 思考・質問

**テーマ**: $ARGUMENTS

> 引数が指定されている場合はその内容を起点に思考・回答します。
> 未指定の場合は直前の会話コンテキストから質問・相談を受け付けます。

## 制約

- **ツール使用禁止** — Read / Edit / Write / Bash / Glob / Grep などのツールを呼び出さない
- **外部アクセス禁止** — WebSearch / WebFetch も使用しない
- 回答はすべて会話コンテキストと自身の知識のみに基づく

## スコープ外の場合

コードの確認・ファイル参照が必要と判断した場合は、回答を試みず以下のように伝える:

> 「このスキルではコードベースを参照できません。`/planning` または直接確認を試してください。」

## 方針

- 質問の意図を正確に捉えてから回答する
- 不明点があれば仮定を明示して回答する
- 「わからない」ときは正直にそう伝え、推測と事実を区別する
- 複数の選択肢があるときはトレードオフを示す

## 出力形式

質問の種類に応じて形式を使い分ける:

| 質問の種類 | 形式 |
|-----------|------|
| 単純な質問・概念説明 | 箇条書きなし、2〜4文の直接回答 |
| トレードオフ・比較 | 選択肢ごとに見出しを立てて比較 |
| 設計・アーキテクチャ相談 | 結論 → 根拠 → 代替案の順 |
| アイデア整理・壁打ち | 論点を箇条書きで整理してから議論 |

簡潔に、ただし必要な深さで答える。

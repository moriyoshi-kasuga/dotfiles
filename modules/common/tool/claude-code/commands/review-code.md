# コードレビュー

**対象**: $ARGUMENTS

> - **ファイルパス指定**: そのファイルを対象にする
> - **`git diff` / `HEAD` 等の指定**: 差分を対象にする
> - **未指定**: `git diff HEAD` の未コミット変更全体を対象にする

## 制約

- **コードの変更は行わない** — 読み取りのみ
- 指摘はすべて「問題の説明 + 改善提案」をセットで行う
- スタイル・好みの問題と実際のバグ・リスクを区別する

## レビュー観点

- **正確性** — ロジックのバグ・エッジケース・境界値の漏れ
- **安全性** — インジェクション・認証漏れ・機密情報の露出など OWASP Top 10
- **可読性** — 命名・関数の長さ・複雑度・コメントの必要性
- **保守性** — 重複・責務の分離・変更容易性
- **パフォーマンス** — 不要な計算・N+1・メモリリーク
- **テスト** — テストの網羅性・エッジケースの考慮

## 出力形式

### 良い点

[優れているコード・設計を列挙]

### 指摘事項

各指摘は以下の形式で記載する：

- **[重要度: 高/中/低]** `ファイル名:行番号` — 問題の説明 → 改善提案

重要度の基準：

- **高**: バグ・セキュリティ問題・データ損失リスク
- **中**: パフォーマンス問題・保守性の低下・テスト漏れ
- **低**: 可読性・スタイル・軽微な改善

### サマリー

[全体的な評価と、優先して対応すべき点]

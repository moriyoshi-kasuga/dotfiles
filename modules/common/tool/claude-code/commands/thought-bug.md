# バグ原因特定

バグの根本原因を体系的に特定します。症状から根本原因まで掘り下げ、修正方針を提示します。

## 調査プロセス
1. **症状の確認** — エラーメッセージ・再現手順・発生条件を整理する
2. **仮説の列挙** — 考えられる原因を最低3つ挙げる（決めつけない）
3. **証拠の収集** — ログ・スタックトレース・関連コード・git log を調査する
4. **仮説の絞り込み** — 証拠をもとに可能性の低い仮説を除外する
5. **根本原因の特定** — 直接原因だけでなく、なぜそれが起きたかまで掘り下げる
6. **修正方針の提案** — 修正案・影響範囲・テスト方針を提示する

## 調査観点
- 最近のコード変更（`git log --oneline`, `git diff`）
- 外部依存（ライブラリ・API・DB）のバージョンや変更
- 環境・設定の違い（dev/staging/prod）
- 競合状態（Race condition）やタイミング問題
- エッジケース・境界値・null/undefined の扱い
- 型の不一致やキャストのミス
